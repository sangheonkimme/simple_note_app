오너: 제품 / UX / 엔지니어링 (공동)

## 1) 개요

iOS/Android용 하이브리드 앱으로 만들어지는 빠르고 친숙하며 오프라인 우선인 노트 앱이다. 사용자는 아이디어, 체크리스트, 이미지를 기록하고 폴더 + 태그로 정리할 수 있다. v1에서는 모든 데이터가 기기에만 저장되며, 향후 버전에서 클라우드 동기화가 지원될 예정이다.

### 1.1 목표

- 2탭 이내 캡처: FAB 또는 위젯으로 빠르게 노트 작성
- 오프라인 전용(v1): 모든 노트 관련 기능은 인터넷 연결 없이 완벽하게 동작
- 간단한 정리: 폴더 + 태그, 빠른 검색
- 신뢰 & 프라이버시: 기기 암호화, 투명한 데이터 사용
- 즐거운 UX: 부드러운 애니메이션, 접근성, 빠른 반응

### 1.2 비목표 (v1)

- 데이터 동기화 및 클라우드 백업
- 팀 협업(공유, 댓글)
- 풍부한 문서 포맷(표, 코드 블록)
- 웹 클라이언트

## 3) 핵심 사용자 스토리 (v1 필수)

1. 사용자는 제목/본문이 있는 텍스트 노트를 만들 수 있다.
2. 사용자는 체크리스트 노트를 만들고 항목을 토글할 수 있다.
3. 사용자는 노트에 이미지를 첨부할 수 있다(1–5장, 각 10MB 이하).
4. 사용자는 노트를 폴더에 넣고 태그를 붙여 정리할 수 있다.
5. 사용자는 제목, 본문, 태그 전반을 로컬에서 검색할 수 있다.
6. 사용자는 노트를 핀/즐겨찾기하여 상단에 고정할 수 있다.
7. 사용자는 노트를 아카이브하거나 삭제할 수 있으며, 휴지통에서 30일 내 복원 가능하다.
8. 사용자는 이메일과 비밀번호 또는 Google 계정으로 가입/로그인할 수 있다. (계정 정보만 존재, 노트 데이터와 연동되지 않음)

Should-Have (v1.1): 데이터 동기화, 다크 모드, Face/Touch 잠금, 다중 선택, MD/PDF 내보내기, 리마인더
Nice-to-Have: 드로잉/필기, 오디오 노트, 웹 클리퍼, 협업

## 4) 정보 구조 & 내비게이션

- 온보딩/소개 → 가입/로그인 (이메일, Google)
- 홈: 인사말, 4개 기본 폴더(개인/학업/업무/기타), 하단 탭바(Home, Search, Add, Settings)
- 폴더(예: Personal): 카드 그리드 목록, FAB “새 노트 추가”
- 노트 상세/에디터: 제목, 본문, 체크리스트 모드 토글, 태그, 첨부, 핀, 폴더 선택
- 검색: 쿼리, 필터(폴더, 태그, 유형, 날짜)
- 설정: 계정, 외관, 프라이버시, 정보

## 5) 기능 요구사항

### 5.1 노트 (로컬 전용)

- 텍스트 & 체크리스트 노트 CRUD
- 로컬 자동 저장(1초마다 또는 blur 시)
- 첨부: 이미지 추가/제거
- 핀/즐겨찾기 불리언, 아카이브 플래그, 휴지통 30일 TTL

### 5.2 정리 (로컬 전용)

- 폴더(시스템 기본 + 사용자 정의)
- 태그: 노트당 다중 태그
- 정렬: updatedAt(기본), createdAt, 제목

### 5.3 검색 (로컬 전용)

- 즉시 로컬 검색
- 범위: 제목, 본문, 체크리스트 항목, 태그

### 5.4 인증 & 계정

- 이메일 + 비밀번호 가입 및 로그인 (외부 API 연동)
- Google 소셜 로그인
- v1에서는 사용자 계정 정보만 관리하며, 노트 데이터와 연동되지 않습니다.

### 5.5 동기화 & 오프라인 (v1.1+)

- v1에서는 모든 데이터가 기기에만 저장됩니다. (100% 오프라인 동작)
- 데이터 동기화는 향후 버전에서 구현될 예정입니다.

## 7) 기술 스택 (하이브리드)

- 프레임워크: Flutter
- 상태 관리: Riverpod
- 로컬 DB: Isar
- 인증: 외부 REST API (계정 관리용)
- 백엔드: 커스텀 (향후 데이터 동기화를 위해 사용 예정)
- 크래시/분석: Firebase Crashlytics + Analytics
- CI/CD: fastlane, GitHub Actions

———

## 8) 데이터 모델 (v1, 로컬)

- **User**: v1에서는 사용되지 않음
- **Folder**: id, name, system (bool), sortOrder, createdAt, updatedAt
- **Note**: id, folderId, title, body, type, pinned, archived, trashedAt, updatedAt, createdAt
- ... (기존 모델에서 userId 제거)

———

## 9) API (v1)

**인증 API** (계정 관리 전용)

POST   /v1/auth/signup     # 이메일 회원가입 (body: email, password, name)
POST   /v1/auth/login      # 이메일 로그인 (body: email, password)
POST   /v1/auth/google     # Google 소셜 로그인/가입 (body: id_token)

**데이터 동기화 API (향후 구현)**
- 노트, 폴더, 태그 등의 CRUD API는 v1.1 이상에서 구현 예정
